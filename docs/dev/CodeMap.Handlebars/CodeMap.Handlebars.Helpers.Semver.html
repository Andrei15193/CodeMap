Remarks

The Semver mapping takes into account all 4 parts of a Version object.

Major Major
Minor Minor
Revision Patch
Build This is translated into Prerelease if the value is part of the following intervals: Basically a 4-digit code where the first one indicates the type of prerelease (1 - alpha, 2 - beta, 3 - rc) where the remaining 3 digits is the number of the prerelease (2001 means beta1)

Example

The following template will generate the Semver expression for the given parameter. 

<p>{{Semver version}}</p>

If the current context exposes a version property that is a Version equal to 1.0.0.0, the output will be as follows: 

<p>1.0.0</p>
